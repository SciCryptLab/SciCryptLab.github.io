The problem with the system today

Reproducible data is harder to produce than it looks.

Real reproducibility means knowing, for every figure in the paper, exactly which script ran on exactly which input with exactly which parameters, and being able to rerun the whole thing months or years later. That trail does not exist by default: scripts drift, parameters get tuned by hand, intermediate files get renamed, the laptop that ran the analysis gets replaced, and the chain of what produced what quietly breaks before the paper is even submitted. Reconstructing it after the fact is detective work — and it always lands on a deadline.

Meanwhile, the rules tightened. The DFG's Guidelines for Safeguarding Good Research Practice require data underlying published findings to be retained "for an appropriate period — generally at least ten years." NIH 2023 Data Management policy requires explicit DMS plans. The ERC has been heading the same way for years. Every funder wants the same thing: real, durable, machine-readable provenance — and the current answer is a folder of files and a researcher who has to remember all of it themselves.

How SciCrypt solves it

Run your script. Get the provenance free.

SciTrace drives Git and git-annex for you. Every time you execute a script through SciTrace, the run is recorded as a special [SciTrace:Run] commit — with the exact command, inputs, outputs, and exit status baked into the commit message as machine-readable JSON. Your dataflow grows a structured history without you writing a single extra line of documentation.

How it shows up in your day.

No new habits. Just the existing one, captured.

1

Create a dataflow once.

Your project gets a clean directory: raw_data/, scripts/, results/, plots/ — plus a metadata.json and a README. Git-annex handles the large binaries.

2

Run your scripts through SciTrace.

SciTrace executes the command and writes the run record back into the commit. The next time you (or anyone) opens the dataflow, the Vis.js lineage graph already shows which inputs produced which output via which script. AI lifecycle analysis flags any data manipulation that smells implicit or undocumented.

3

Get an integrity score before you submit.

When you link a SciTrace dataflow to a SciVerify submission, every figure is automatically scored on traceability and reproducibility. Your manuscript carries a single data integrity score — and anything weak shows up before the editor sees it.

Why open source matters here.

Provenance is leverage. Locking your provenance into a vendor's database is leverage handed back.

  • Self-hostable. Run SciTrace on your own server or laptop. Your manuscripts and raw data never leave your infrastructure.
  • Inspectable. Every rule, every score formula, every check is readable code in the repo. No black boxes.
  • Forkable. Your discipline has unusual needs? Bend the rubric.
  • Funder-friendly. Open data + open tools is what they're asking for.
