What it is

When an author links a SciTrace dataflow to a SciVerify submission, every figure is automatically checked against two verifiable signals — traceability and reproducibility — drawn from the git repository and its provenance history.

What it can do

  • Traceability check (15% weight). Verifies that every script and data file referenced by a figure is git-tracked and committed — no uncommitted edits, no untracked dependencies. A clean git status earns a full score.
  • Reproducibility check (25% weight). Searches the commit history for [SciTrace:Run] provenance records. Confirms that the run's inputs still match their hashes, that the producing script hasn't been edited after the run, and that outputs haven't been hand-replaced.
  • Deep re-execution (optional). An opt-in deep check re-runs the recorded command inside a cached environment and byte-compares the outputs. Auto-invalidates when scripts or inputs change — never stale.
  • Per-figure scoring. Each linked figure gets its own traceability + reproducibility score, weighted 37.5 / 62.5 and banded green / amber / red. Any figure below 70% lights up before the editor sees it.
