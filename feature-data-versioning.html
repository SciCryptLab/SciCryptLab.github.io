What it is

When you create a dataflow, SciTrace lays down a standard layout and a metadata.json, then versions it with its own VCS layer over Git + git-annex. A .gitattributes rule routes small text files (scripts, CSVs, JSON) into Git and large binaries (images, datasets, PDFs) into git-annex automatically — so history stays fast and nothing is too big to track.

What it can do

  • Structured by default. Every project gets raw_data/ scripts/ results/ plots/, a metadata.json, and a README — the shape reviewers and funders expect.
  • Right backend per file. Text in Git, binaries in git-annex, decided automatically — no manual add juggling.
  • Restore anything. Browse the full history and bring back a deleted or earlier version of any file from any commit, all in the browser.
  • Save a stage in one click. Snapshot your current work as a commit without opening a shell.
  • Self-hostable. The repository lives on your machine or server. Your raw data never has to leave your infrastructure.
Why it matters

Funder mandates (DFG, ERC, NIH) increasingly require durable, machine-readable data retention. A SciTrace dataflow is versioned and structured from the first commit — the documentation is the byproduct, not a separate chore.
