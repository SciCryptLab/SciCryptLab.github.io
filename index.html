Research integrity is eroding.
Reproducibility is the first to suffer.

The research ecosystem was never built for the threats it now faces.

Paper mills at industrial scale

Automated fabrication factories produce thousands of fake papers monthly. Existing detection tools catch some — but operate in silos, and the scale is growing faster than any single system can handle.

AI-written content in submissions

LLMs generate fluent prose that humans can't reliably distinguish from authentic research. Hallucinated citations, fabricated results, and AI-generated discussion sections slip past peer review undetected.

FAIR mandates from funders

DFG, ERC, and NIH now require data to be findable, accessible, interoperable, and reusable for a decade. Most labs lack the infrastructure to comply — and reviewers have no way to verify.

Two tools, one integrity chain.

One tool tracks where every result came from.
The other turns it into an integrity signal editors can trust.
Together, they form a continuous integrity chain.

Open source

SciTrace · for researchers

A native Git + git-annex dataflow with a standard layout (raw_data, scripts, results, plots). Every script run gets recorded as a provenance commit. Vis.js shows you the lineage. FAIR by construction. Produces a verifiable provenance package that SciVerify uses to score integrity — open source, self-hostable, your data, your machine.

SciTrace features → See it in the demo → Source code →

SciVerify · for editorial integrity

Consumes the SciTrace provenance package and runs automated checks: references against Crossref, AI-text indicators across every section, authors against OpenAlex, and dataflow scored on traceability, reproducibility, and manipulation transparency. Editors see one integrity number, with the full evidence one click away.
